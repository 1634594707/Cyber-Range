# 🛡️ Cyber Range 网络安全靶场 — 架构设计文档

## 1. 服务器资源配置

| 配置项   | 规格                | 用途说明                     |
| -------- | ------------------- | ---------------------------- |
| OS       | Debian 12.11        | 稳定、安全、长周期支持       |
| CPU      | 8核                 | 支撑并发挑战 + Docker 编排   |
| 内存     | 8GB                 | CTFd + DB + 10~15 挑战容器   |
| 系统盘   | 50GB SSD            | 操作系统 + 平台代码 + 日志   |
| 数据盘   | 50GB SSD            | Docker 数据 + 数据库持久化   |

| 区域     | 中国香港            | 免备案，国际连通性好         |

## 3. 技术栈选型

### 3.1 平台核心

| 组件       | 技术选型          | 版本     | 选择理由                           |
| ---------- | ----------------- | -------- | ---------------------------------- |
| CTF 平台   | **CTFd**          | 3.7.x    | 社区标准，插件丰富，Docker 原生支持 |
| Web 服务器 | Gunicorn          | 21.x     | 生产级 WSGI，多 worker             |
| 反向代理   | Nginx             | 1.26     | 高性能，TLS 终止，WAF 模块         |
| 数据库     | PostgreSQL        | 16       | JSON 支持，ACID，性能优于 MySQL     |
| 缓存       | Redis             | 7.x      | 会话存储，排行榜缓存               |
| 容器引擎   | Docker + Compose  | 最新稳定 | 挑战隔离，快速部署                 |

### 3.2 监控与安全

| 组件     | 技术选型   | 用途                       |
| -------- | ---------- | -------------------------- |
| 监控     | Prometheus | 指标采集                   |
| 可视化   | Grafana    | 实时仪表盘                 |
| 日志     | Loki       | 日志聚合                   |
| 入侵防护 | Fail2Ban   | SSH/Web 暴力破解防护       |
| 防火墙   | nftables   | Debian 12 原生防火墙       |
| 证书     | Let's Encrypt | 免费 TLS 证书           |

## 4. 网络架构

### 4.1 Docker 网络规划

```
网络名称       子网段         用途
─────────────────────────────────────────
ctfd-net      172.20.0.0/24  CTFd + DB + Redis（内部通信）
web-challenges 172.21.0.0/24  Web 安全类挑战容器
pwn-challenges 172.22.0.0/24  PWN/二进制挑战容器
isolated      172.30.0.0/24  完全隔离沙箱（无外网访问）
```

### 4.2 端口映射

```
外部暴露（Nginx 后）:
  :443  → CTFd Web（HTTPS）
  :80   → 301 → :443

内部端口（仅 VPC 内网）:
  :8000   CTFd Gunicorn
  :5432   PostgreSQL
  :6379   Redis
  :3000   Grafana

SSH 管理:
  :22022 (修改默认端口)
```

## 5. 数据持久化方案

```
│   ├── uploads/
│   └── logs/

```
